## Part 1 — what happened

### Phase 1 — Orient and plan

The session began from a handoff document (`_meta/HANDOFF.md` § "SESSION 2 HANDOFF") that defined four task groups in priority order: Crossplane v1→v2 corpus sweep, OPS-layer trim, threading all logged rulings into spec/plan files, and my-scope cleanup items. The DECISIONS-LOG had every ruling recorded but nothing yet applied to the 238 spec/plan files.

Before touching any files, three sample reads were done: the ADR-0041 spec (to understand the v1 claim model's concrete vocabulary), the B4 spec (the heaviest file with 39 "claim" occurrences), and the Canon glossary (to understand which XRD names needed renaming). This grounded the subagent briefs in real file content rather than abstract rules.

ADR structural work was done manually before launching subagents:

This was done synchronously rather than delegated because it was structural (file moves, new files) and needed to land before subagents could reference the new ADR.

Six parallel agents were launched covering non-overlapping file sets:

|---|---|---|

Agents 2 and 5 hit stream idle timeouts after 45+ minutes. Both had already written substantial work to disk by the time they timed out; the work was visible in `git status` and committed via checkpoint commits.

**Checkpoint commits were critical.** Because multiple agents write to disk concurrently and the sandbox is ephemeral, checkpoint commits were made at each completion notification — not at the end. This meant two timed-out agents still had their completed work preserved.
